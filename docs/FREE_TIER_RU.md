# Бесплатный трекер на тысячи пользователей — что реально

## Зачем кэш (не «экономия ради жадности»)

Кэш — это **снимок портфеля на момент времени**, сохранённый на сервере (и в браузере).

| Без кэша | С кэшем |
|----------|---------|
| Каждый заход = новый scrape/API (5–20 с) | Первый заход = загрузка, повторный = **<1 с** |
| 10k юзеров = 10k× нагрузка на DeBank/Jina | Один кошелёк обновляется раз в N минут для **всех** |
| Бан по IP, 502, таймауты | Стабильный UX |

### Если баланс изменился через минуту

- **Свежий кэш (по умолчанию 5 мин)** — показываем снимок без нового запроса. Для DeFi это нормально: позиции не меняются каждую секунду.
- **Устаревший кэш (до 1 ч)** — отдаём сразу старый снимок, **в фоне** подтягиваем новый (кнопка «Обновить» = принудительно сейчас).
- **Кнопка «Обновить»** — всегда обходит кэш (`refresh=1`).

Настройка в `config.json`:

```json
"cache": {
  "portfolio_fresh_sec": 300,
  "portfolio_stale_sec": 3600,
  "portfolio_max_sec": 86400
}
```

### Что кэшируется

| Данные | Где | Зачем |
|--------|-----|--------|
| Полный портфель (токены + DeFi) | `.cache/portfolio/` | главный экран |
| Revert (диапазоны LP) | `.cache/revert/` | APY, range |
| Сессия пользователя | `sessionStorage` | не дёргать API при F5 |

История графика строится из **текущих объёмов × CoinGecko**, не live on-chain.

## Почему нельзя «просто скопировать DeBank» бесплатно

DeBank — это **data company**:

- свои ноды и индексеры по сотням протоколов;
- база позиций всех пользователей **заранее**;
- команда на парсеры новых контрактов.

«Скопировать сайт» = либо **их API/scrape** (лимиты, ToS), либо **построить то же** (месяцы/годы, не $200).

Бесплатный scrape (Jina → debank.com) для **10k DAU**:
- заблокируют IP;
- 20+ секунд на кошелёк;
- юридически серая зона.

**Вывод:** для MVP на сотни юзеров scrape ок. Для 10k — нужна **своя инфраструктура данных**, не копипаст HTML.

---

## Почему on-chain «всё сам» тоже не бесплатно на 10k

On-chain без индексера = на каждый кошелёк:

- RPC по 5–10 сетям;
- логи NFT / MasterChef / Aave / Compound…;
- декодинг каждого протокола отдельно.

Один полный скан = **минуты** и тысячи RPC-вызовов.  
10k пользователей × 3 раза в день = **миллионы** вызовов → платные RPC или свой узел ($$$).

То, что у тебя уже работает (~$56k портфель за ~8 с) — это **агрегатор DeBank (scrape)**, не чистый on-chain.

## Бюджет $200 и 10k пользователей — честная математика

| Вариант | 10k DAU | $200 |
|---------|---------|------|
| DeBank API units | ~2000 полных refresh **всего**, не в день | хватит на тест, не на 10k |
| Scrape DeBank | IP ban за часы | нет |
| On-chain full | RPC лимиты + 10+ мин/юзер | нет |
| **Scrape/API + жёсткий кэш 15–60 мин** | возможно, данные «отстают» | да, как компромисс |
| **Свой лёгкий слой** (Etherscan free + top-3 протокола) | 1–3k юзеров, не DeBank-parity | частично |

**Идеал «10 сек + как DeBank + $0» для 10k одновременно — нереалистичен.**  
Реалистичная цель на $200:

1. **До ~500–2000 активных кошельков/день** с кэшем 10–30 мин (текущий стек).
2. Постепенно **свои парсеры** только для Uniswap V3 + Aave + Compound (80% объёма многих кошельков).
3. Etherscan free (5 req/s) + публичные RPC для **wallet-only** между полными обновлениями.

## План «упереться и сделать» по этапам

### Этап 0 — сейчас (уже есть)
- Агрегатор `source=auto` → DeBank scrape → кэш → Revert.
- Stale-while-revalidate: быстрый снимок + фоновое обновление.

### Этап 1 — бесплатно усилить (1–2 недели)
- [ ] TTL кэша 15–30 мин на PA (`config.json`).
- [ ] Очередь refresh (не больше N scrape параллельно).
- [ ] Wallet-only preview: Etherscan multichain за 2–3 с, DeFi — из кэша.
- [ ] Метрики: сколько scrape/день в `.cache/api-debug.log`.

### Этап 2 — «достаточно хорошо» без DeBank (1–3 мес)
- Парсеры: Pancake V3 (Base/Arb), Compound V3, Aave V3.
- Цены: DefiLlama / CoinGecko free.
- DeBank scrape только fallback.

### Этап 3 — 10k (если продукт взлетит)
- Свой воркер + Postgres snapshots.
- Платный RPC **или** $200 DeBank units **только** для «тяжёлых» кошельков.
- Остальное on-chain.

---

## Итог одной фразой

**Кэш** — чтобы 10k человек не убили источник данных и видели ответ за секунду; свежесть — компромисс (минуты), кнопка «Обновить» — выход.  
**Копировать DeBank бесплатно на 10k** — нельзя без своей базы; **$200** хватит на умный MVP с кэшем, не на безлимит DeBank-quality для всех каждый раз.
