# RPC-режим (прод)

## Про «15 минут кэша»

**Это не время ожидания пользователя.** Раньше на сервере лежал снимок 15 мин — если *тот же* адрес уже сканировали, ответ был мгновенный. Новый кошелёк всё равно ждал полный ончейн-скан.

Сейчас: **кэш ~90 с** только от повторного F5 того же адреса. Первый заход = **всегда свежий скан** (`refresh=1`).

## Сколько ждёт новый человек (холодный кошелёк, PA + public RPC)

| Что сканируем | Оценка |
|---------------|--------|
| Pancake V3 **Base** (farm + LP), только DeFi | **~1–2 мин** |
| Pancake **Base + Arb + BSC** | **~3–6 мин** |
| + Uniswap V3 те же сети | **+2–4 мин** |
| + Aave / Compound / Morpho lending | **+30–90 с** на протокол |
| «Всё как DeBank» без индексатора | **10–20+ мин** — нереалистично за «секунды» |

Узкое место: **getLogs** по 10k блоков на бесплатных RPC. Alchemy paid + субграфы → можно целиться в **15–30 с** для LP.

## UX сейчас

1. **Быстрый проход** (`quick=1`): только LP (Pancake на base/arb/bsc) — показываем позиции.
2. **Фон**: кошелёк (ETH, USDC, …) догружается.

## Сети Pancake

Код умеет **Base, Arbitrum, BSC** (NFPM + MasterChef). Ethereum у Pancake V3 в нашем реестре нет — там другие деплои; при необходимости добавим адреса.

## RPC-провайдер

[Alchemy](https://www.alchemy.com/) в `config.json` → `rpc_urls.base` (и arb/bsc). Публичные ноды — для dev; прод на 10k — платный RPC.

## DeBank

Только dev (`train-protocol`, scrape). В проде **не используется**.
